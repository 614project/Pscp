# PS/CP 언어 명세 v0.2

## 1. 범위

이 문서는 문제 풀이 및 경쟁 프로그래밍에 특화된 프로그래밍 언어의 구문(syntax), 정적 규칙(static rules), 실행 모델(execution model)을 정의한다. 이 언어는 표현식 지향(expression-oriented)이며, 중괄호로 블록을 구분하고, C#으로의 트랜스파일을 통해 컴파일되며, 간결한 알고리즘 코드에 최적화되어 있다.

달리 명시하지 않는 한, 이 문서는 규범적(normative)이다.

---

## 2. 설계 목표

1. 알고리즘적 의도를 간결하게 표현할 수 있을 것.
2. 들여쓰기에 의미적으로 의존하지 않는 중괄호 기반 문법을 사용할 것.
3. 명시적인 지역 가변성(local mutability)을 제외하고는 기본적으로 불변(immutable)일 것.
4. `break`, `continue`, `return`을 포함한 명령형 제어 흐름을 실용적으로 지원할 것.
5. 컬렉션 표현식, 범위, 집계(aggregation), 경량 반복 구문을 일급(first-class)으로 지원할 것.
6. 경쟁 프로그래밍 환경에 적합한 입출력 단축 표현을 내장할 것.
7. 선택적 헬퍼 런타임 지원을 활용한 가독성 높은 C# 코드로 낮출(lower) 수 있을 것.

## 3. 소스 형식

### 3.1 문자 집합

소스 파일은 유니코드 텍스트로 작성된다.

### 3.2 공백

공백(whitespace)은 필요한 경우 토큰을 구분한다. 개행(newline)은 일반적으로 의미론적으로 중요하지 않으며, 단 한 줄 구문(one-line constructs)에서 명시적으로 언급된 경우는 예외이다.

### 3.3 주석

다음 주석 형식이 예약되어 있다:

```txt
// 단일 행 주석
/* 여러 행 주석 */
```

주석 문법은 어휘적(lexical)이며, 주석은 파싱에 참여하지 않는다.

### 3.4 구문 구분

구문(statement)은 개행, 중괄호, 또는 세미콜론으로 구분할 수 있다.

세미콜론은 선택 사항이다. 세미콜론은 한 줄에 여러 구문을 작성할 때 사용할 수 있다.

```txt
var x = 0; x += 1; += x
```

후행 세미콜론(trailing semicolon)도 허용된다.

```txt
let var mut rec
if then else

break continue return

```

### 4.2 특수 토큰

```txt
= += -= *= /= %=
== != < <= > >= <=>
&& || ^ !
.. ..< ..=
|> <|

```

토큰 `..`은 문맥에 따라 범위 연산자(range operator)를 나타내거나, 컬렉션 표현식 내부에서 스프레드 요소(spread element)를 나타낼 수 있다.

## 5. 어휘 범주

### 5.1 식별자

식별자는 유니코드 문자 또는 밑줄로 시작하며, 이후에 문자, 숫자, 밑줄이 올 수 있다.

단일 밑줄 `_`은 예약된 버림 토큰(discard token)이며, 일반적인 식별자가 아니다.

### 5.2 정수 리터럴

정수 리터럴은 기본적으로 십진수이다.

예시:

```txt
0
1
42
1000000
1L
```

후행 `L`은 `long` 타입을 나타낸다.

### 5.3 부동소수점 리터럴

부동소수점 리터럴은 C# 스타일의 어휘 규칙을 따른다.

예시:

```txt
3.14
0.5
1e9
```

### 5.4 문자 리터럴

문자 리터럴은 작은따옴표를 사용한다.

```txt
'a'
'\n'
```

### 5.5 문자열 리터럴

문자열 리터럴은 큰따옴표를 사용한다.

```txt
"hello"
"a b c"
```

## 6. 타입

### 6.1 내장 스칼라 타입

다음 스칼라 타입이 내장되어 있다:

- `int`
- `long`
- `double`
- `decimal`
- `bool`
- `char`
- `string`

### 6.2 튜플 타입

튜플 타입은 괄호로 묶인 쉼표 구분 타입 목록으로 작성된다.

```txt
(int, int)
(long, int, string)
```

튜플의 차수(arity)는 타입의 일부이다.

### 6.3 배열 타입

1차원 및 중첩 배열 타입을 지원한다.

```txt
int[]
int[][]
(string, int)[]
```

### 6.4 제네릭 컬렉션 타입

명세에서 인식하는 제네릭 컬렉션 형식은 다음과 같다:

```txt
List<int>
LinkedList<int>
```

추가적인 제네릭 타입 지원은, 명시적으로 지정되지 않는 한 구현 정의(implementation-defined)이다.

### 6.5 크기 지정 선언 타입

크기 지정 선언(sized declaration)은 선언 위치의 배열에서 허용된다.

```txt
int[n] arr
int[n][m] grid
```

이 형식은 선언 구문이며, 일반적인 타입 표현식이 아니다.

## 7. 바인딩, 가변성 및 초기화

### 7.1 기본 불변성

모든 바인딩은 `mut` 또는 `var`로 선언되지 않는 한 불변이다.

### 7.2 선언 형식

#### 7.2.1 명시적 타입 지정 불변 선언

```txt
int x = 3
string s = "abc"
```

#### 7.2.2 타입 추론 불변 선언

```txt
let x = 3
let s = "abc"
```

#### 7.2.3 명시적 타입 지정 가변 선언

```txt
mut int x = 0
mut string s = ""
```

#### 7.2.4 타입 추론 가변 선언

```txt
var x = 0
var s = ""
```

### 7.3 다중 선언

선언 이름의 수와 제공된 값의 수가 일치하면 다중 선언이 허용된다.

```txt
int a, b = 1, 2
let x, y = f()
var i, j = 0, 0
```

우변은 튜플 값이거나 구문적으로 쉼표로 구분된 값일 수 있다.

### 7.4 초기화되지 않은 선언

#### 7.4.1 가변 스칼라 및 참조

초기화되지 않은 선언은 명시적 타입 지정 가변 바인딩에만 허용된다.

```txt
mut int x;
mut string s;
mut int[] arr;
```

해당 바인딩은 타입의 기본값으로 초기화된다.

#### 7.4.2 크기 지정 배열

크기 지정 배열은 초기화 값을 생략할 수 있다.

```txt
int[n] a;
int[n][m] dp;
```

이러한 선언은 배열을 할당하고, 모든 원소를 원소 타입의 기본값으로 초기화한다.

#### 7.4.3 금지된 형식

다음은 컴파일 타임 오류이다:

```txt
int x;
string s;
let a;
var b;
```

### 7.5 버림(Discard)

토큰 `_`은 버림(discard)을 나타낸다.

허용되는 위치:

- 대입의 좌변
- 선언 대상
- 구조 분해(destructuring) 대상
- 람다 매개변수

예시:

```txt
_ = f()
let _ = g()
(a, _, c) = foo()
_ => 1
(a, _, c) => a + c
```

버림은 값으로 참조할 수 없다.

```txt
let x = _      // 오류
```

## 9. 컬렉션 표현식

### 9.1 구문

컬렉션 표현식은 대괄호로 묶인다.

```txt
[]
[1, 2, 3]
[1, x + 1, y]
```

### 9.2 기본 결과 타입

#### 9.2.1 선언 문맥

`let`을 사용하는 선언 문맥에서, 원소 타입을 추론할 수 있는 경우 컬렉션 표현식은 기본적으로 배열 타입이 된다.

```txt
let a = [1, 2, 3]    // int[]
```

#### 9.2.2 명시적 타입 문맥

선언 대상 타입이 명시적인 경우, 컬렉션 표현식은 해당 타입으로 변환된다.

```txt
int[] a = [1, 2, 3]
List<int> b = [1, 2, 3]
LinkedList<int> c = [1, 2, 3]
```

#### 9.2.3 우변값(R-value) 문맥

순수 우변값 문맥에서, 컬렉션 표현식은 대상 타입 또는 오버로드 해결 규칙에 따라 이터러블(iterable) 또는 구체적인 컬렉션으로 낮춰질 수 있다. 구현은 관찰 가능한 의미론을 보존해야 한다.

### 9.3 범위의 자동 확장

컬렉션 표현식 내에서, 범위 원소는 자동으로 확장된다.

```txt
[0..<5]           // [0,1,2,3,4]
[1, 2, 0..<5]     // [1,2,0,1,2,3,4]
```

자동 확장은 범위 표현식에만 적용된다.

### 9.4 스프레드 원소

스프레드 원소는 이터러블을 주변 컬렉션 표현식으로 명시적으로 확장한다.

```txt
let a = [3, 4, 5]
let b = [1, 2, ..a, 6, 7]
```

규칙:

1. 스프레드 구문은 컬렉션 표현식 내부에서만 유효하다.
2. 스프레드의 피연산자는 이터러블이어야 한다.
3. 스프레드는 반복 순서를 보존한다.

### 9.5 빌더 형식

컬렉션 표현식은 컴프리헨션(comprehension) 유사 빌더를 포함할 수 있다.

```txt
[0..<n -> i do i * i]
[arr -> x do f(x)]
```

빌더는 각 반복에 대해 `do` 표현식이 평가된 결과를 원소로 하는 컬렉션을 생성한다.

## 10. 범위 표현식

### 10.1 구문

이 언어는 다음 범위 형식을 정의한다:

```txt
1..10
0..<n
0..=n
10..-1..0
```

### 10.2 의미론

- `a..b` : `a`부터 `b`까지의 포함 범위
- `a..<b` : `a`부터 `b`까지의 범위 (`b` 제외)
- `a..=b` : 명시적 포함 형식
- `a..step..b` : 도달 가능한 경우 끝점을 포함하는 스텝(step) 범위

### 10.3 이터러블 여부

범위 표현식은 이터러블 값이다. 다음 위치에서 사용할 수 있다:

- `for` 루프
- `->` 반복
- 컬렉션 표현식
- 집계 표현식
- 함수 호출

## 11. 멤버 접근, 인덱싱 및 튜플 투영

### 11.1 멤버 접근

```txt
obj.member
obj.method
```

### 11.2 인덱싱

배열, 문자열, 인덱싱 가능한 컬렉션에는 C 스타일 인덱싱을 사용한다.

```txt
arr[i]
grid[r][c]
text[i]
```

### 11.3 튜플 투영

튜플 투영은 점(.) 뒤에 양의 정수 리터럴을 붙여 작성한다.

```txt
p.1
p.2
p.3
```

규칙:

1. 튜플 투영은 1부터 시작한다(1-based).
2. 점의 우변은 정수 리터럴이어야 한다.
3. 동적 튜플 인덱싱은 지원하지 않는다.

예시:

```txt
(int, int, int) p = (6, 1, 4)
= p.2
```

## 12. 연산자

### 12.1 단항 연산자

- `+x`
- `-x`
- `!x`
- `not x`

`not`은 불리언 부정의 키워드 별칭이다.

### 12.2 산술 연산자

- `*`
- `/`
- `%`
- `+`
- `-`

### 12.3 비교 연산자

- `<`
- `<=`
- `>`
- `>=`
- `==`
- `!=`
- `<=>`

우주선 연산자(spaceship operator) `<=>`는 피연산자의 순서 관계에 따라 `-1`, `0`, `1` 중 하나를 반환한다.

### 12.4 논리 연산자

기호 형식과 키워드 형식 모두 지원된다.

- 논리곱(conjunction): `&&`, `and`
- 논리합(disjunction): `||`, `or`
- 배타적 논리합(exclusive or): `^`, `xor`

키워드 형식은 기호 형식의 정확한 별칭이다.

### 12.5 대입 연산자

- `=`
- `+=`
- `-=`
- `*=`
- `/=`
- `%=`

대입은 선언 구문에 의해 달리 지정되지 않는 한 가변 대상이 필요하다.

### 12.6 파이프 연산자

- `|>`
- `<|`

파이프 연산자는 표현식 위치에서 함수 적용 및 순서 지정에 사용된다.

## 13. 연산자 우선순위 및 결합성

우선순위가 높은 것부터 낮은 것 순으로:

1. 후위(postfix)
   - `f(x)`
   - `a[i]`
   - `obj.member`
   - `p.1`

2. 전위(prefix)
   - `+x`
   - `-x`
   - `!x`
   - `not x`

3. 곱셈(multiplicative)
   - `*`
   - `/`
   - `%`

4. 덧셈(additive)
   - `+`
   - `-`

5. 범위(range)
   - `..`
   - `..<`
   - `..=`

6. 비교(comparison)
   - `<`
   - `<=`
   - `>`
   - `>=`
   - `<=>`

7. 동등성(equality)
   - `==`
   - `!=`

8. 논리 AND
   - `&&`
   - `and`

9. 논리 XOR
   - `^`
   - `xor`

10. 논리 OR
    - `||`
    - `or`

11. 파이프(pipe)
    - `|>`
    - `<|`

12. 대입(assignment)
    - `=`
    - `+=`
    - `-=`
    - `*=`
    - `/=`
    - `%=`

범위 표현식은 덧셈 표현식보다 약하게 결합된다. 예를 들어:

```txt
0..<n + 1
```

은 다음과 같이 파싱된다:

```txt
0..<(n + 1)
```

## 14. 반환 의미론

### 14.1 명시적 반환

`return` 구문은 함수, 람다 블록, 또는 블록 표현식 내부에서 사용할 수 있다.

```txt
return expr
return
```

`return`은 가장 안쪽에서 감싸는 함수 또는 람다 본문을 종료한다.

### 14.2 암묵적 반환

블록은 `return` 없이도 값을 생성할 수 있다.

블록은 마지막 구문이 다음 조건을 모두 만족하는 표현식 구문인 경우에만 암묵적으로 반환한다:

1. 블록의 마지막 구문일 것,
2. 세미콜론으로 종료되지 않을 것,
3. 단독 호출 표현식(bare invocation expression)이 아닐 것.

### 14.3 단독 호출(Bare Invocation)

단독 호출은 최상위 형식이 함수 또는 메서드 호출인 표현식 구문이다.

단독 호출의 예시:

```txt
foo x
foo(x)
obj.method x
obj.method(x)
```

마지막 위치의 단독 호출은 그 값을 암묵적으로 반환하지 않는다.

### 14.4 단독이 아닌 마지막 표현식

다음 마지막 표현식은 암묵적으로 반환된다:

```txt
a <=> b
not hashset.Add 10
foo(x) == true
if ok then 1 else 2
x + y
```

### 14.5 값 버리기

값을 의도적으로 무시하려면 버림 대입을 사용해야 한다.

```txt
_ = hashset.Add 10
```

## 15. 함수 및 람다

### 15.1 함수 선언

함수는 쉼표로 구분된 명시적 매개변수 목록을 사용한다.

```txt
int add(int a, int b) {
    a + b
}
```

### 15.2 재귀 함수

재귀는 기본적으로 비활성화되어 있다. 함수는 `rec`로 선언된 경우에만 자기 자신을 재귀적으로 참조할 수 있다.

```txt
rec int fact(int n) {
    if n <= 1 then 1 else n * fact(n - 1)
}
```

`rec`가 아닌 함수에서의 자기 참조는 컴파일 타임 오류이다.

### 15.3 람다 표현식

람다는 `=>`를 사용한다.

```txt
x => x + 1
(acc, x) => acc + x
(acc, x) => {
    let y = acc + x
    y % mod
}
```

람다 매개변수는 버림이 될 수 있다.

```txt
_ => 1
(a, _, c) => a + c
```

### 15.4 호출 구문

#### 15.4.1 괄호를 사용한 호출

표준 호출 구문이 지원된다.

```txt
f(x)
g(x, y)
```

#### 15.4.2 공백 구분 호출

호출 시 괄호와 쉼표를 생략할 수 있다.

```txt
add 1 2
f x
g (x + 1) y
```

규칙:

1. 공백 구분 호출은 일반적인 다중 인수 호출에 대한 표면 구문이다.
2. v0.2에서는 커링(currying)이나 부분 적용(partial application)을 암시하지 않는다.
3. 복잡한 인수는 괄호가 필요하다.
4. 파서는 우선순위 규칙 하에서 최대 유효 인수 그룹화를 우선한다.

권장되는 원자적 인수 형식:

- 식별자
- 리터럴
- 인덱싱 표현식
- 멤버 접근 표현식
- 괄호로 묶인 표현식
- 람다 표현식
- 컬렉션 표현식

## 19. 빠른 반복 구문

### 19.1 구문 형식

### 19.3 의미론

`->`는 이터러블 값에 대한 반복을 도입한다.

- 단일 변수 형식은 원소를 바인딩한다.
- 인덱스 형식은 0부터 시작하는 인덱스와 원소를 바인딩한다.

`->` 구문은 v0.2에서 반복 구문에만 배타적으로 예약되어 있다.

## 20. 집계 표현식

집계 표현식은 반복 본문으로부터 결과를 계산한다.

### 20.1 기본 형식

```txt
min { for i in 0..<n do a[i] - b[i] }
sum { for x in arr do x }
count { for x in arr do x % 2 == 0 }
```

### 20.2 필터 형식

```txt
count { for x in arr where x % 2 == 0 do 1 }
sum { for x in arr where ok(x) do x }
```

내장 집계기(aggregator)의 집합은 구현 정의이지만, 적어도 `min`, `max`, `sum`, `count`는 포함되어야 한다.

## 21. 한 줄 구문

### 21.1 `then`

### 21.2 `do`

`do`는 다음 위치에서 유효하다:

- `for ... do ...`
- `while ... do ...`
- `xs -> x do ...`
- 집계 본문

1. 하나의 표현식 또는 단순 구문 본문만 허용된다,
2. 본문은 개행 또는 구문 종결자에서 끝난다,
3. 중첩된 여러 줄 본문에는 중괄호가 필요하다.

## 25. 정적 오류

다음은 컴파일 타임 오류이다:

1. 불변 바인딩에 대입,
2. `_`을 값으로 사용,
3. `rec`가 아닌 함수에서의 자기 재귀 참조,
4. 여러 구문에 걸쳐 있는 잘못된 형식의 한 줄 `then` 또는 `do` 본문,
5. 튜플이 아닌 값에 대한 튜플 투영,
6. 리터럴이 아닌 인덱스를 사용한 튜플 투영,
7. 지원되지 않는 자동 입출력 형태,
8. 루프 외부에서 `break` 또는 `continue` 사용,
9. 모호한 괄호 없는 복잡한 인수를 가진 공백 구분 호출 사용,
10. 초기화되지 않은 불변 바인딩,
11. 컬렉션 표현식 내부의 유효하지 않은 스프레드 피연산자.

## 26. 특수 파싱 규칙

### 26.1 출력 구문

`= expr`과 `+= expr`은 구문 시작 위치에 나타날 때만 출력 구문으로 파싱된다.

### 26.2 입력 선언

우변 표현식이 없는 `T x =` 형식의 선언은 불완전한 대입이 아닌 입력 단축 표현으로 파싱된다.

### 26.3 스프레드 vs 범위

컬렉션 표현식 내부에서:

- 쉼표 또는 컬렉션 시작 뒤에 오는 `..expr`은 스프레드를 나타내고,
- `a..b`, `a..<b`, `a..=b`, `a..step..b`는 범위 표현식을 나타낸다.

### 26.4 공백 구분 호출

파서는 우선순위 규칙에 따라 인수 경계가 명확한 경우에만 공백 구분 호출을 허용해야 한다. 복잡한 인수 표현식을 명확히 구분하려면 괄호가 필요하다.

### 26.5 블록의 마지막 표현식

블록의 마지막 구문만 암묵적 반환에 참여할 수 있다.

## 27. 트랜스파일 모델

이 언어는 특정 단일 백엔드에 독립적으로 명세되어 있으나, 참조 구현은 C#을 대상으로 한다.

### 27.1 낮추기(Lowering) 원칙

1. 관찰 가능한 의미론을 보존한다.
2. 가독성 높은 C# 코드 생성을 우선한다.
3. 직접적인 C# 변환이 어색할 때는 문법적 설탕(syntactic sugar)을 헬퍼 메서드로 낮춘다.
4. 의미론이 일치할 때는 C# 내장 기능을 사용한다.

### 27.2 낮추기 예시

```txt
int[] parent = [0..<n]
```

는 다음으로 낮춰질 수 있다:

```csharp
int[] parent = Enumerable.Range(0, n).ToArray();
```

```txt
let b = [1, 2, ..a, 6, 7]
```

는 연결(concatenation) 후 실체화(realization)로 낮춰질 수 있다.

```txt
(arr[i], arr[j]) = (arr[j], arr[i])
```

는 C# 튜플 교환 대입으로 직접 낮춰질 수 있다.

## 30. 적합성(Conformance)

구현이 이 명세에 적합하려면:

1. 본 문서에서 정의한 모든 구문적으로 유효한 프로그램을 허용하고,
